RAG Vector DB(Qdrant) SPO 검색 자동화 감사 효율화
Problem

🔎 감사 현장에서의 자료 탐색 비효율

1) SPO 기반 감사자료 관리의 현실

  • 각 감사팀이 SPO를 활용해 대규모 자료를 관리하지만, 필요한 파일을 찾는 데 많은 시간 소요
  • 팀/프로젝트마다 폴더 구조가 달라, 신규 투입 인력은 위치 파악에 어려움
  • 파일명·키워드에 크게 의존하는 검색 방식 → 검색 누락 및 중복 자료 증가
  • 감사 수행 중 “자료 찾기”가 야근을 유발하는 대표적인 업무 중 하나

2) 고객사 내부회계·결산 대응에서도 동일 Pain Point

  • 내부회계 용역 대응 시, 필요한 자료를 찾는 데 정말 큰 시간이 소요되는 것으로 파악
  • 결산·내부통제 관련 자료가 여러 부서·여러 시스템에 흩어져 있음
  • “필요한 자료를 빠르게 찾고 싶다”는 니즈는 어느 회사든 공통적으로 존재

📌 핵심 Pain Point 요약

  • 자료는 충분히 있지만, 찾는 데 시간이 많이 걸리는 문제
  • 사람의 기억·노하우에 의존한 비표준적인 검색 프로세스
  • 감사팀·고객사 모두에게 공통적으로 발생하는 구조적 비효율

👉 이 문제를 해결하면, 감사 품질·속도·조직 학습효과까지 모두 동시에 개선 가능

Solution

🤖 RAG 기반 SPO 자연어 탐색기

“SPO 안의 모든 자료를, 자연어 한 문장으로 찾아줍니다.”

① SPO 문서 → 벡터 DB(Qdrant)화

  • SPO에 저장된 문서의 본문, 파일명, 메타데이터 추출
  • 임베딩(Embedding)을 통해 의미 기반 검색이 가능한 벡터로 변환
  • Qdrant Vector DB에 저장하여, 대용량에도 빠른 검색 성능 확보

② RAG 기반 자연어 질의

  • 사용자가 “작년 재고자산 평가 관련 보고서 파일 찾아줘”처럼 자연어로 질문
  • LLM이 질의를 이해하고, Qdrant에서 관련 문서를 의미 기반으로 검색
  • 가장 관련도가 높은 파일 후보군과 함께, 요약 정보·메타데이터 제공

③ SPO URL 연동 & 다운로드

  • 검색된 문서의 SPO URL을 즉시 제공
  • 사용자는 결과 리스트에서 원하는 문서를 클릭 → 바로 열기/다운로드
  • 별도의 폴더 탐색 없이, “질문 → 파일”로 바로 연결

📂 SPO Connector (Microsoft Graph API)
🧠 Embedding + Qdrant Vector DB
💬 LLM 기반 RAG 검색 파이프라인
🔗 SPO URL Resolver & 다운로드 연동
Architecture

🧩 기술 아키텍처 Overview

*이 슬라이드는 PPT에서 다이어그램으로 시각화해도 좋고, 아래 텍스트를 설명용으로 그대로 활용해도 됩니다.*

데이터 플로우

  • Microsoft Graph API
    Graph API를 통해 SPO에 접근, 파일 본문 및 경로, 메타데이터 추출
  • Embedding Pipeline
    텍스트 전처리 후, 임베딩 생성 및 Qdrant에 저장
  • Qdrant Vector DB
    문서별 벡터 + 메타데이터 관리, 필터·조건 검색 지원

검색 & 응답 플로우

  • RAG Engine
    자연어 질의 분석 → 벡터 검색 → LLM 응답 생성
  • SPO URL Resolver
    검색된 문서의 SPO 실제 URL/다운로드 경로 매핑
  • UI Layer
    Web, Teams Bot 등에서 챗봇 형식으로 질의/응답 제공

Differentiation

🔥 차별화 포인트

기존 방식의 한계

  • 키워드 중심 검색 → 파일명/태그에 과도한 의존
  • SPO 기본 검색 기능만으로는 전체 텍스트 기반 정교한 검색이 어려움
  • 일반 LLM은 사내 실제 문서에 직접 접근할 수 없어, “진짜 파일”을 바로 제공하지 못함

RAG 기반 SPO 탐색기의 강점

  • 의미 기반(Semantic) 검색으로, 표현이 달라도 동일 주제의 문서를 찾아줌
  • Qdrant 기반 벡터 DB로 대용량 SPO 문서에도 확장성 확보
  • SPO URL 바로 연동으로, “검색 결과 → 즉시 다운로드”까지 한 번에 연결

👉 단순 “검색”을 넘어, 실제 작업에 바로 쓸 수 있는 결과를 제공하는 것이 핵심

  • 엑셀 등 주요 문서 확장자 형식 지원

3) 내부 활용도 확대
